## 一、先认清这不是第一次改文案

本站文案有过一次顶层重做（2026-07-03，见 [copy-redesign-handoff.md](copy-redesign-handoff.md)
与 [copy-principles.md](copy-principles.md)）。那一轮的诊断是：**第一版把“确定性算法、
指纹校验”当卖点头牌，写出了工具味**——于是把首屏摆到情绪侧，产出了现在的
“慌的时候，先起一卦”。

现在朋友仍然第一眼看不懂。**这不是上一轮改错了，是改得只剩一半**：情绪头牌立住了，
但品类认知的地基没跟上。本轮迭代的分寸感就在这里，一句话钉死：

> **情绪内核不动，补认知地基。绝不允许摆回“机制头牌”的老路——那条路 07-03 已经
> 用一整轮重做证明走不通。**

**结构性缺口三样**（比措辞更硬的问题）：

1. **品类失锚**：首屏没有任何一个给产品定性的名词。诗意标题悬在半空。
2. **零产品视觉**：Hero 是纯文字 + 抽象着色器；全站唯一一张产品截图（result.webp）
   在折叠线以下。陌生人没有任何具象锚点知道“这东西长什么样”。
3. **“为什么是你”缺席**：用户的旧体验是把问题粘贴给通用 AI（这正是本产品的立项
   起点），首屏没有一句话回答“我为什么专门下一个 App”。

**执行时必须先核的两个事实雷点**（开工第一问，找 owner 拍板）：

- FAQ“支持哪些平台？——都上架了……安卓去 Google Play”：Android 在架的 v101 对
  纯 v3 后端已无法起卦（每卦失败），且此前有过“撤下安卓不实陈述 → owner revert”
  的历史（`7ea17e4` → `edd30da`）。本轮必须与 owner 对齐口径后统一处理，不许绕开。
- en 页场景卡以“尋／緣／業”单字呈现：对英文用户是纯装饰性汉字，加重“看不懂”。

## 三、理论标尺（本轮改版的判据，全部带出处）

### 梁宁侧（三部精读结论，取最锋利的七把刀）

1. **特性 vs 属性 + 无抗猪肉**（《真需求》PART TWO）：特性是自己花成本做出来的点，
   属性是用户对它的反应。“他只交付了核心价值，而没有管理用户共识。连用户共识的
   第一步——用户感知的设计，他都没有做。”本站病根的书面定义：有价值点、用户感受
   不到。
2. **KANO 分层**（《真需求》PART TWO）：起卦不出错对占卜 App 是**必备属性**——达标
   不加分、出错才减分，不是拉动选择的期望属性。“你承诺的，如果能确定性交付，用户
   就满意；不能确定性交付，就是智商税。”
3. **意识即防御**（《产品思维》005）：“一个产品如果引发用户启动意识，让用户思考，
   某种意义上，就是在推开用户。”凡要陌生人“琢磨一下”的首屏表述一律不合格。
4. **打孔机 vs 墙上的洞**（《产品思维》013）：用户不需要打孔机，需要墙上有个洞。
   机制话（我怎么保证）一律翻译成用户拿到的确定性（每一卦都算得对、道长一直记得你）。
5. **用户价值公式**（《产品思维》023）：新体验 − 旧体验 − 替换成本。旧体验 = 把问题
   粘贴给通用 AI。首屏必须一眼说清净增量，并让替换成本显得可忽略（报两个数字就行）。
6. **高概念低感知**（《增长思维》035）：卖点在概念上成立、用户当场无体感 = 无从认知
   更无从传播。三条创始人论点全部命中此病。
7. **口碑 = 替用户说好那句话**（《产品思维》022 +《增长思维》012）：“口碑也是替消费
   者设计一句他要说的话，然后让他说出来。”凡用户不会脱口而出转述给朋友的句子，
   不配占第一眼。

### 外部最佳实践侧（2026-07-21 联网调研，一手 URL）

1. **定位五要素链，hero 是终点不是起点**（April Dunford）：竞品替代（含“什么都不做”
   和“粘给通用 AI”）→ 独特属性 → 价值 → 谁最在乎 → 品类锚。hero 必须“品类 + 价值”
   一起说清。https://www.aprildunford.com/post/a-quickstart-guide-to-positioning
2. **Grunt Test 三问**（StoryBrand）：你提供什么／对我有什么好／怎么获得——穴居人
   五秒能答才合格。自检句：“我们卖的是 ___”，填不满或要靠行话 = 不合格。
   https://jareddees.com/grunt-test/
3. **标题写成描述不是口号**（Julian Shapiro）：“若访客只读这一句，他知道你卖什么吗？”
   https://www.julian.com/guide/startup/landing-pages
4. **具体名词胜过形容词、可证伪、唯一句**（Harry Dry）：“给你此刻能做的一件小事”
   （可检验）胜过“带来内心平静”（空泛）；“报两个数字，按固定方法起一卦，不是随机
   抽签”是只有本产品能说的唯一句。https://marketingexamples.com/copywriting/tips
5. **outcome-first，AI 不当主角**：“Users don't buy AI — they buy outcomes.”
   标题给转变，机制退副标。https://www.valtorian.com/blog/ai-product-mistakes-2026
6. **情绪结果 + 具体方法并列句式**（Headspace／Calm 范式）：结果讲给心，方法讲给脑。
   https://www.headspace.com
7. **把玄讲成可触摸**（Co-Star 范式）：越玄的品类越要具体可操作的语言去神秘化。
   https://www.costarastrology.com
8. **五秒测试操作规程**（Lyssna）：5–10 名目标人群、看 5 秒遮住、问“这页是关于
   什么的”，≥80% 说对主旨 = 通过；收敛性误解定位硬伤。
   https://www.lyssna.com/guides/five-second-testing/
9. **单 CTA 独大**：首屏一个主动作视觉主导，其余弱化。
   https://www.landy-ai.com/blog/landing-page-hero-section

### 竞品镜鉴（既有沉淀盘点，2026-07-21；主力在 hachimi-ios/docs/research/）

- **最值得偷的一招——测测的对比话术**：“DeepSeek 不认识你另一半，测测 AI 能基于
  对方星盘解读这段关系。”（KB `wiki/entities/cece-xinyan.md`）用一句对比同时完成
  “为什么不用通用 AI”与“我是谁”——正是本站缺的那句。哈基米的同构句自然是：
  通用 AI 不记得你上回问过什么，道长记得。
- **FateTell** tagline“Be Your Own Fate Teller”，coaching 前置先接情绪再排盘；
  它吃“决策焦虑”，与自家“情绪陪伴”是两条线，学句式不学定位。
- **问真八字 26 屏逐屏拆解**（iOS 仓 `docs/research/2026-07-02-competitor-wenzhen-bazi-teardown.md`）：
  卖点写“你能得到什么”而非功能罗列；术语点击即弹白话；“此刻”即时起卦零门槛；
  分人群讲不同价值叙事。**需避开**：文墨天机式“零解读零情绪的专家计算器”冷感墙。
- **自家已有三条候选定位陈述**（iOS 仓 `docs/research/2026-06-12-product-positioning-zhenxuqiu.md`
  §三，2026-06-12 写就，本轮改 hero 前必读），其中第二条与本文档六节话术同源。
- **Apple 语境**：fortune-telling 是 4.3 饱和类目，“猫皮人设 + 梅花起卦 + 情绪陪伴”
  是过审差异化盾——官网品类锚措辞与商店口径保持一致，别为了“一眼懂”写成裸“算命”。
- **已知沉淀缺口**：海外玄学 App（Co-Star／CHANI／Sanctuary）官网 hero 原文横评
  尚无专门沉淀（本文档三节的 Headspace／Calm／Co-Star 首屏句是 2026-07-21 现抓，
  已部分补上）；执行会话若需更多样本可现抓，抓完按 research-sediment 落盘。

## 四、三条创始人论点的裁决（对外话术指南）

owner 向朋友介绍时惯用三条论点。逐条裁决 = 工程事实核验（2026-07-21 盘点了后端
与两端代码、评测、文档）× 梁宁框架。**共同病灶：三条全是供给侧机制话，答的是
“我们怎么造”，朋友问的是“这对我是什么”。**

### 论点 1 “通用 AI 起卦可能错，我们用代码保证不错” → 事实成立，但当不了头牌

- **工程事实（可验证）**：起卦核是纯函数（`hachimi-backend/src/engine/meihua.ts`），
  无随机无时钟，《观梅》古例金标 + SHA-256 校验 + iOS/后端三端逐字节对拍全部在测试
  里锁死。另有 LLM 自行起卦的对照实验（`experiments/cast-stability.ts`），但一致率
  数字未提炼成对外可引的口径。
- **梁宁裁决**：必备属性（起卦对是底线不是卖点）＋高概念低感知（圈外人从不知道
  “起卦还会错”，感知不到这个痛）＋打孔机语言。copy-principles 教训第 1 条早已立过：
  **确定性是信任底座，不是营销头牌**。
- **对外可说的版本**（放深层页与口头介绍，不放首屏）：“算卦是确定的，解读是生成的，
  模型动不了卦象也动不了吉凶。”讲故事版：“你拿 ChatGPT 求卦，它每次起的卦可能
  不一样；这里起卦那一步是代码按古法算的，跟老师傅排出来的一字不差。”

### 论点 2 “专用记忆比通用 AI 记忆更聚焦专业” → 内核最强，措辞全错

- **工程事实（形态可验证，比较级无实测）**：端上“道长的记忆”真实在架——每卦蒸馏
  卦忆、受控主题词表、召回至多 3 条宁缺毋滥、P@3 校准测试、记忆只进解读不进起卦由
  字节对拍锁死。但“比 ChatGPT 记忆更聚焦”从未做过对照实测，比较级说不出口。
- **梁宁裁决**：这是三条里唯一贴着立项原点的（当初的洞察就是“通用助手给得了排盘，
  给不了‘记得你上次问过什么’的陪伴”）——**是最接近口碑的真源，但“专用记忆／聚焦
  专业”是参数话术**，对首访陌生人是无感知属性（他还没有任何记忆可言）。它是第二卦
  之后才成立的价值。
- **对外可说的版本**：“道长记得你上回问的事，一卦接一卦地陪你。”首屏可点一笔这个
  情感承诺（它是耐久事实，记忆系统已随 1.1.0 全量出货），深讲放“怎么玩”之后。

### 论点 3 “道长判例训练调试，harness 达到顶级大模型水平” → 站不住，停用

- **工程事实（不成立，且被项目自己的决策反证）**：唐道长判例采集表的批注列**全空**，
  判例从未进入任何 prompt 或评测集；北极星 NS-8 明确锁定“道长经验降级为锦上添花、
  飞轮必须自转、解除关键路径上的外部人依赖”。“训练”一词也不准确——零 fine-tune，
  实际是 prompt 分层 + 评测 harness 工程。“达到顶级大模型水平”零对照证据：全部评测
  横评只在 Flash 档位之间，从未与任何顶级档位模型对比过。
- **梁宁裁决**：匪兵甲（把情绪价值产品拉进与主角的能力数字竞赛）＋逼近智商税表述
  （解读水平不可确定性交付）＋方向性错误（《增长思维》036：智能卖点该讲“能”——
  在你这件事上真的管用，不该讲“智”——我和大模型一样聪明）＋逻辑自我消解（“那我
  为什么不直接用那个顶级模型？”）。
- **对外可说的版本**：“裸模型直接问会跑偏——会乱预测、会丢人设；我们把该定死的
  用代码定死，50 条三语金标每改必测，红线一条不许破。它不比大模型聪明，它在
  ‘认真陪你问这一卦’这件事上管用。”

## 五、执行任务书（按序，SDD）

**第 0 步 · 内容-位置矩阵**（继承 2026-07-03 铁律：先审布局再审句子）：把“是什么／
给谁／解决什么／怎么做到／凭什么信／现在做什么”六问 × 全站位置（hero／场景卡／
功能卡／highlight／principles／methodology／FAQ）画成矩阵，每条论述只允许一个
权威位置。当前已知复位方向：品类锚上首屏；确定性下沉信任区（最多一两处）；记忆
翻译成情感承诺；“免费无广告”按耐久口径继续留 FAQ 不进 hero。

**第 1 步 · Hero 改版**（唯一 writer 串行改稿）：

- 补品类锚：首屏必须出现能定性的具体名词（参考方向：“AI 道长”“占卜 App”“梅花
  易数”至少其一，以口语方式融入，不堆术语）。
- 标题过“只读这一句知道你卖什么”检验；副标用“情绪结果 + 具体方法”并列句式。
- 真实产品截图进首屏（现有 result.webp 或新截图；解决零具象锚点问题）。
- 单 CTA 主导（按平台显示的商店徽章为主动作，其余弱化）。
- “凭什么是你”一句话进首屏或紧邻第二屏（记忆的情感版为首选：它是差异化真源）。
- 红线全数继承：不预测不改命不转运；“起”不用“算”；确定性必带时辰；禁 AI 味词表
  与标点红线（见 [copy-principles.md](copy-principles.md) 第六节，全文有效）。

**第 2 步 · 全页复位与事实修正**：按矩阵搬运各 section；处理两个事实雷点（安卓
口径找 owner 拍板；en 场景卡去装饰汉字）；en 侧同步全部改动（Flesch-Kincaid ≤ 8，
只留 hexagram 一个特殊词）。

**第 3 步 · 验证（把“朋友说看不懂”正式化）**：按 Lyssna 规程做五秒测试——5–10 名
目标人群（会为情绪价值买单、对玄学半信半疑的朋友，不是同事），看首屏 5 秒后遮住，
问“这一页是关于什么的／它提供什么／第一印象”。**通过线：≥80% 能说出“陪你问事／
理心事的 AI 道长 App”方向的答案**；若收敛性误解指向同一句，定点修那一句再测。
另做口碑一句话测试：听完介绍的人能否把产品转述给下一个人。

**工具链**：文案抽 .md 跑 `scan-ai-taste.sh` 红线归零；clean-context reviewer
四焦点评审至多两轮；`npm run lint && npm run typecheck` 门禁；push main 即
CF Pages 自动部署。
